# LLM Conveyors Chrome Extension

Chrome Manifest V3 extension that autofills ATS application forms with a user's LLM Conveyors profile, highlights job-posting keywords, and streams agentic generation results into a side panel. Built with WXT, React, TypeScript, Tailwind v4, consuming the `ats-autofill-engine` npm package.

## Architecture

WXT + React + TypeScript + Tailwind v4 + `ats-autofill-engine` dependency.

## Absolute Invariants

1. **Structured logging only**: NEVER `console.*`. Use `createLogger(scope)` from `src/background/log.ts` (D11). Enforced by `validate:grep-gates`.
2. **No network in UI entrypoints**: `fetch` is forbidden in `entrypoints/content/**`, `entrypoints/popup/**`, `entrypoints/sidepanel/**`, `entrypoints/options/**`. All network goes through background via `sendMessage`.
3. **No cross-adapter imports**: `src/ats/<vendor>/**` MUST NEVER import from `src/ats/<other-vendor>/**`. Each adapter is a sealed bundle.
4. **Taxonomy is internal**: `skill-taxonomy` is consumed only by backend endpoints; extension code MUST NEVER import or reference it. Enforced by grep.
5. **Hexagonal core boundary**: `src/core/**` MUST NEVER import DOM APIs (`document`, `window`, `HTMLElement`), `chrome.*`, or any Node/WXT runtime. Pure TypeScript only.
6. **Structured logger boundary (D11)**: every log call through the `Logger` interface; debug gated on non-production build mode.
7. **Every module ships `blueprint.ts`**: any module in `src/background/**` or `src/content/**` with business logic owns a `blueprint.ts` conforming to `ModuleBlueprint` (see `src/_blueprints/blueprint.types.ts`). Adapter modules in `src/ats/<vendor>/**` ship their own. Orphan modules without a blueprint fail `validate:blueprints`.

## Commands

```bash
# Dev
pnpm dev                 # WXT dev server (chrome target by default)
pnpm dev:firefox         # Firefox target

# Build
pnpm build               # Production build
pnpm build:firefox       # Firefox production build
pnpm zip                 # Zip the built extension for Web Store upload

# Compliance (single gate; runs all checks)
pnpm compliance          # typecheck + lint + test + validate:blueprints + validate:grep-gates + validate:protocol-schema

# Individual checks
pnpm typecheck           # tsc --noEmit
pnpm lint                # eslint + prettier check
pnpm test                # vitest run
pnpm validate:blueprints # AST + sourceRef + handler coverage
pnpm validate:grep-gates # forbidden-token scan
pnpm validate:protocol-schema   # regenerates schema in memory, diffs committed

# Dev-only harness
pnpm tinker              # Vite dev server at localhost:5174 for protocol + adapter tinkering

# Full rigor gauntlet (before PR)
pnpm rigor:full          # pnpm compliance && pnpm validate:protocol-schema
```

## Debugging

Native-tools-only (no MCP server in the extension repo). Typical incident flow:

| Scenario | How |
|----------|-----|
| Service worker is unreachable | `chrome://extensions` -> inspect views -> service worker devtools console |
| Content-script scan misfires | Open target page devtools -> Sources -> Content scripts tab |
| Message handler throws | Search `[llmc-ext:messaging]` in bg service-worker console |
| Protocol schema mismatch | `pnpm validate:protocol-schema` locally; CI fails on drift |
| Blueprint drift | `pnpm validate:blueprints`; read violation location + blueprint source |
| ATS adapter fails on real page | Run `pnpm tinker`, load vendor fixture + live DOM snapshot, observe `FormModel` |

## Infrastructure

- **Target runtime**: Chrome 120+ (Manifest V3 service worker).
- **Build tool**: WXT 0.20 (`pnpm dlx wxt@latest init`).
- **Store distribution**: Chrome Web Store (production), Firefox Add-ons (optional).
- **Backend contract**: `llmconveyors.com` API (OAuth refresh flow, credits, keywords, generation).
- **Engine dependency**: `ats-autofill-engine` on npm (MPL-2.0), consumed through explicit sub-entries (`/greenhouse`, `/lever`, `/workday`, `/dom`, `/profile`).
- **CI**: GitHub Actions, single workflow `.github/workflows/compliance.yml` runs `pnpm compliance` on every PR.
- **Quality substrate**: `.claude/rules/` + `src/_blueprints/` + `scripts/_quality/` + `tests/harness/` + `docs/rigor/` materialized in Phase A0.

## State Preservation

Before ending a session, update `MEMORY.md` with:

- Phases completed since last entry (phase code + commit hash).
- Open blockers that block the next phase (severity + file:line + owner).
- Decisions made this session that are NOT in a plan doc yet (promote to blueprint or decision memo on next pass).
- Next step intended for the next session (one sentence).

Failure to update `MEMORY.md` after a 3+ commit session triggers a Husky `post-commit` warning (installed in A1).
